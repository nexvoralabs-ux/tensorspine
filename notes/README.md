# notes

`derivations.md` is the mathematical core of this project and is meant to be
worked out **by hand on paper first**, then transcribed here — the transcription
is a check that the derivation actually closed, not a substitute for doing it.
`shortcuts.md` is the honesty ledger: every time a rule was looked up or handed
over instead of derived, it gets logged there with the date, so the gap between
"the tests pass" and "I understand why" stays visible.
